# ✨ 추억 가계부 (Memoir Account Book)
> **"단순한 가계부를 넘어, 오늘의 소비와 추억을 한 장의 카드로 남기는 감성 기록 서비스"**

본 프로젝트는 신입 개발자로서 **사용자 중심의 UI/UX 설계**와 **안정적인 백엔드 API 구축**, 그리고 **데이터 시각화** 역량을 증명하기 위해 제작되었습니다. 최근 트렌드인 **'Butter Yellow' 파스텔 테마**와 **모바일 퍼스트 디자인**을 적용하여 실제 앱을 사용하는 듯한 완성도를 자랑합니다.

---

## 🚀 주요 기능 (Key Features)

### 1. 감성적인 데이터 기록 & 관리
- **추억 카드 시스템 (Photo Upload):** 거래 내역 입력 시 '오늘의 기록'과 함께 **실제 사진**을 업로드할 수 있습니다. 메인 화면에서 폴라로이드 사진 느낌의 감성적인 카드로 시각화됩니다. (UUID 기반 파일 관리 및 정적 리소스 핸들링 적용)
- **실실시간 지출 분석:** `Recharts`를 활용하여 카테고리별 지출 비중을 파이 차트로 제공합니다.
- **예산 관리 시스템:** 이번 달 목표 예산을 설정하고, 지출 현황을 게이지 바(`LinearProgress`)로 실시간 추적합니다. (예산 80% 초과 시 경고 색상 변경 로직 적용)
- **지능형 카테고리 분리:** 수입(월급, 보너스 등)과 지출(식비, 교통 등)의 성격에 맞게 카테고리를 분리하여 입력 편의성을 높였습니다.

### 2. 고도화된 UI/UX
- **추억 검색 서비스 (Search):** 키워드(일기 제목, 본문, 카테고리) 기반의 **서버 사이드 검색 엔진**을 구축하여 과거의 기록을 빠르게 찾을 수 있습니다. (500ms 디바운싱 적용으로 성능 최적화)
- **자주 쓰는 가계부 (Favorites):** 반복되는 지출/수입은 '즐겨찾기' 템플릿으로 등록하여 클릭 한 번으로 빠르게 입력할 수 있습니다. 각 카테고리에 맞춘 전용 아이콘 시스템을 도입했습니다.
- **모바일 앱 스타일 레이아웃:** 폰 프레임 디자인과 하단 플로팅 탭 바(Floating Bottom Bar)를 통해 네이티브 앱과 유사한 UX를 제공합니다.
- **트렌디한 테마:** '버터 옐로우'와 '크림' 컬러를 조합한 파스텔 톤 디자인 시스템을 구축했습니다.
- **Glassmorphism:** 입력 폼에 반투명 유리 효과를 적용하여 현대적이고 입체적인 디자인을 구현했습니다.
- **금액 포맷팅:** 숫자 입력 시 실시간으로 천 단위 콤마(`,`)를 적용하고 음수 입력을 차단하는 등 사용자 편의성을 극대화했습니다.

### 3. 안정적인 백엔드 & 보안
- **BCrypt 암호화:** 사용자의 비밀번호를 해시 알고리즘(BCrypt)으로 암호화하여 DB 보안성을 확보했습니다.
- **JWT 인증 시스템:** JSON Web Token을 활용하여 보안 인증 체계를 구축하고, Axios 인터셉터를 통해 프론트엔드와 안전하게 통신합니다.
- **1:1 엔티티 설계:** `Transaction`(거래)과 `Diary`(일기) 테이블을 1:1 관계로 설계하여 데이터의 독립성과 확장성을 고려했습니다.

## 🛠 Tech Stack

### Frontend
- **React 18**
- **Material-UI (MUI)**: 커스텀 테마 및 디자인 시스템 구축
- **Framer Motion**: 부드러운 리스트 전환 및 컴포넌트 애니메이션
- **Recharts**: 지출 통계 데이터 시각화
- **Axios**: API 통신 및 JWT 인터셉터 처리

### Backend
- **Spring Boot 3.x**
- **Spring Security & JWT**: 보안 및 인증 처리
- **Spring Data JPA**: MySQL 연동 및 ORM 처리
- **Lombok**: 생산성 향상 및 코드 간결화
- **MySQL**: 관계형 데이터베이스 관리

## 📂 Trouble Shooting (문제 해결 경험)

### 1. 프론트-백 API 엔드포인트 불일치 해결
- **문제:** 로그인 시도 시 404/405 에러 발생.
- **원인:** 프론트엔드 요청 경로(`/api/auth`)와 백엔드 컨트롤러 경로(`/api/members`)의 명세가 달랐음.
- **해결:** 개발자 도구의 Network 탭을 활용해 요청 주소를 대조하고, RESTful API 설계 규칙에 맞춰 경로를 통일하여 통신 성공.

### 2. 비밀번호 암호화 및 검증 로직 구현
- **문제:** DB에 암호화되어 저장된 비밀번호와 로그인 시 입력값이 일치하지 않아 인증 실패.
- **해결:** `PasswordEncoder.matches()` 메소드를 활용하여 입력된 평문과 해시된 값을 안전하게 비교하도록 로직 보완.

### 3. 모바일 프레임 내부 하단 바 고정 이슈
- **문제:** `fixed` 포지션 사용 시 하단 바가 폰 프레임 바깥(브라우저 하단)에 위치하는 문제 발생.
- **해결:** `App.js`에서 공통 레이아웃 구조를 잡고 `absolute` 포지션을 활용하여 폰 프레임 내부 하단에 고정함으로써 실제 앱과 동일한 레이아웃 구현.

### 4. 업로드 이미지 접근 권한(403 Forbidden) 해결
- **문제:** 이미지 업로드 후 조회를 시도했으나 `403 Forbidden` 발생.
- **원인:** Spring Security 설정에서 정적 리소스 경로인 `/uploads/**`에 대한 접근을 인증된 사용자만 가능하게 제한했기 때문.
- **해결:** `SecurityConfig.java`의 `requestMatchers`에 `/uploads/**` 경로를 `permitAll()`로 추가하여 공용 접근이 가능하도록 조치.

## ⚙️ 시작하기 (Quick Start)

### Backend
1. `application.properties`에서 MySQL 정보 설정
2. `./gradlew bootRun` 실행 (포트: 8088)

### Frontend
1. `npm install`
2. `npm start` 실행 (포트: 3000)
